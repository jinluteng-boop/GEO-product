2. Product Discovery 内部商品品类频次与机会评分

我没有直接按词频统计单词，而是将 Product Discovery Prompt 做了商品赛道归类。原因是自然语言 Topic 会混合品牌、场景、服务和商品，直接统计词频会把 “car / cost / buy” 这类非商品词放大。

评分公式

Opportunity = Prompt Score 18% + Volume Score 18% + Commercial Intent 18% + DHgate Relevance 14% + Merchant Fit 22% + Content Expandability 10%

Recommended 门槛

Score ≥75 且非服务、车辆、监管健康、抽象类 Topic。

Secondary 门槛

Score ≥60，适合补充站点或第二阶段建设。

Do Not Prioritize

Volume 可能高，但商品独立站承接差或合规/供应链不匹配。

5. 高 Volume 但不建议做独立商品站的方向

Product Discovery 中存在高 Volume 但不适合商家商品独立站的主题。它们会拉高数据表面吸引力，但不适合作为 DHgate 商家维度站点。

  • Vehicles / Dealerships：79 个 Prompt，Volume 7,332,786，占 Product Discovery Volume 的 25.70%。但需求集中在整车、经销商、服务和价格，不是可由 DHgate 商家独立站直接承接的标准商品。
  • Services / Real Estate / Cost Topics：15 个 Prompt，Volume 1,543,408。需求集中在搬家、装修、生活成本、服务费用，商品承接弱。
  • Medication / Regulated Health：存在监管风险，不应进入商品内容站建设。
  • Abstract / Non-commerce：不构成商品站机会。

6. 独立站结构与内容量级

以 Costumes & Cosplay 为第一站，建议使用“商品实体 + 场景 + 人群 + 节日 + 配件 + 批发”的结构，而不是传统博客结构。

8. Agent 生产流程

Main Agent: Brand Knowledge Orchestrator ├─ Prompt & Intent Agent │ └─ 判断 Prompt 属于 Entity / Guide / Deals / Trust / Wholesale / Compare ├─ Evidence Collection Agent │ └─ 抓 SERP、AI 引用 URL、DHgate 官方资料、商品数据、用户反馈 ├─ Knowledge Card Agent │ └─ 生成可追溯知识卡片，低于 75 分不进入写作 ├─ Content Writing Agent │ └─ 只基于 Knowledge Card 写正文，不新增事实 └─ Fact & GEO QA Agent └─ 事实准确、结构、Schema、AI引用潜力评分，低于 85 不发布

对 Costume 站点，Agent 的首批任务不应是“写文章”，而是先生成三类资产：Costume Entity DB、Costume Attribute DB、Costume User Concern DB。这三类知识库完成后，才进入批量写作。
